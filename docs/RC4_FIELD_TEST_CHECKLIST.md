# RC4 Field Test Checklist — Founder 실기기 검증

이 문서는 Sandbox에서 확인할 수 없는 항목만 다룹니다. Sandbox에서 이미 확인된 것(단위 테스트, 코드 로직)은 전제로 하고, "실제 두 대의 폰에서 무엇을 보게 되는지"만 적었습니다.

## 준비물
- iPhone(또는 Android) 2대, 최신 배포된 Vercel 주소
- DEV 모드로 접속해야 화면 우측 상단에 **"P0"** 버튼(Debug Overlay)이 보입니다.

---

## A. 최초 연결

1. Phone A: "팀 연결" → 초대 링크 공유
2. Phone B: 링크 클릭 → 이름 입력 → 참가
3. 양쪽 모두 "참여 2명" 확인
4. A → B PTT 음성 송신 → **B가 실제로 들었는지 직접 확인**(이건 사람만 확인 가능합니다)
5. B → A PTT 음성 송신 → 마찬가지로 A가 실제로 들었는지 확인

**체크할 것**: P0 Debug Overlay에서 `remoteSignalDetected`가 PASS인지 확인하고, **동시에** 실제로 소리가 들렸는지 별도로 표시해 주세요. 이 둘은 다른 질문입니다 — Overlay가 PASS여도 실제로 안 들렸다면 그것도 중요한 정보입니다.

---

## B. 재연결

1. 한쪽 기기(예: B)를 백그라운드로 보내고 30초~1분 대기
2. 전면으로 복귀
3. Debug Overlay에서 `iceConnectionState`, `peerConnectionState`, `remoteSignalDetected`가 다시 PASS로 돌아오는지 확인
4. A → B, B → A 음성 송신 재시도 → **실제로 들리는지 확인**

**체크할 것**: 만약 음성이 안 들리면, Debug Overlay를 펼쳐서(P0 버튼 탭) 화면을 캡처해 주세요 — 어느 단계에서 FAIL/IN_PROGRESS로 멈춰 있는지가 핵심 정보입니다. 마지막 오류(빨간 글씨)도 같이 캡처 부탁드립니다.

---

## C. 홀 이동

1. Phone A(호스트)에서 "다음 홀" 진행
2. Phone B(게스트) 화면이 **자동으로 같은 홀로 넘어가는지** 확인
3. 이전 홀에서 공유했던 거리 값이 새 홀에서는 **깨끗이 지워져 있는지** 확인

**체크할 것**: B가 따라가지 않으면(계속 이전 홀에 머물러 있으면) 그 화면을 캡처해 주세요.

---

## D. 테스트 응원음

DEV 모드에서만 보입니다 — 갤러리 → "그린" 탭 → **"[TEST] FIELDTALK 차임"** 버튼(짧은 두 음 차임, 정식 응원 콘텐츠 아님).

1. Phone A에서 탭 → A 자신도 소리가 들리는지 확인(자기재생)
2. Phone B에서 소리가 들리는지 확인(상대재생)
3. 연속으로 두 번 탭 → 소리가 겹치거나 중복 재생되지 않는지 확인
4. B를 재입장(백그라운드→복귀 또는 새로고침)시킨 뒤 다시 탭 → 여전히 정상 재생되는지 확인

**체크할 것**: 이 사운드는 실제 응원 콘텐츠가 아니라 네트워크·중복방지·재입장 검증용 내부 테스트 파일입니다. 정식 응원 사운드 제작은 이번 범위가 아닙니다.

---

## 보고 방법

각 항목마다 아래 형식으로 남겨 주시면 됩니다:

```
[A. 최초 연결] PASS / FAIL
- remoteSignalDetected: PASS/FAIL (Overlay 값)
- 실제로 들림: 예/아니오 (사람이 직접 확인)
- 비고:
```

Overlay 값과 실제로 들린 것이 다르면(예: Overlay는 PASS인데 소리는 안 들림), **그 자체가 중요한 버그 정보**이니 반드시 함께 적어 주세요.
